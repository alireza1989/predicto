Agentic Loop — Core Pattern
The Anthropic SDK tool-use loop powering every agent
# agents/base.py — the heart of the system

while iterations < self.max_iterations:
  response = self.client.messages.create(
    model=self.model,        # "claude-sonnet-4-20250514" or "claude-opus-4-20250514"
    max_tokens=self.max_tokens,
    system=self.system_prompt, # Role-specific instructions
    tools=self.tool_schemas,  # JSON tool definitions (name, description, input_schema)
    messages=self.messages,   # Full conversation history
  )

  if response.stop_reason == "end_turn":  # Agent decided it's done
    break

  for block in response.content:
    if block.type == "tool_use":
      result = tool.execute(block.input)  # Run the Python function
      messages.append({
        "role": "user",
        "content": [{"type": "tool_result", "tool_use_id": block.id, "content": result}]
      })
      # Loop continues — Claude sees result, decides next action
Claude Code Concepts — What We Used vs. What's Available
Mapping Predicto to the Claude / Anthropic ecosystem

Anthropic Python SDK — Tool Use

✓ USED — Core Pattern

Every agent is powered by anthropic.Anthropic().messages.create() with tools= parameter. Claude sees tool schemas, decides which to call, receives results, and iterates. This is the manual agentic loop — the foundational building block.

agents/base.py → Agent class, 27 tools across 5 agents

Multi-Agent Orchestration

✓ USED — Sequential Pipeline

5 specialized agents run in sequence, each with its own system prompt, tools, and model. The orchestrator (main.py) passes context summaries between agents. Each agent autonomously decides how to use its tools.

main.py → run_pipeline() chains Data → Feature → Meta-Scientist → Eval → Report

Multi-Model Strategy

✓ USED — Sonnet + Opus

Claude Sonnet for routine agents (data, features, eval, report) — fast and cost-effective. Claude Opus for Meta-Scientist — needs deeper reasoning to design experiments, interpret results, and decide what to try next.

config.yaml → agent_model vs scientist_model

Tool Definitions (JSON Schema)

✓ USED — 27 Custom Tools

Each tool is defined with name, description, and input_schema (JSON Schema). The SDK sends these to Claude, which returns structured tool_use blocks with validated inputs. Our Tool class wraps Python functions with schema metadata.

agents/base.py → Tool dataclass with execute() method

System Prompts (Role Engineering)

✓ USED — Per-Agent Personas

Each agent has a tailored system prompt defining its role, expertise, and workflow. The Meta-Scientist prompt specifies experiment order and scientific method. The Eval Agent prompt defines promotion criteria. This is how we give each agent its "personality."

Each agent file defines SYSTEM_PROMPT with specific instructions

Conversation Memory (In-Context)

✓ USED — Message History

Each agent maintains its full message history within a run. The Meta-Scientist uses this to remember which experiments it already ran and plan the next one. Cross-agent memory is handled via SQLite (experiment_log table) and passed summaries.

Agent.messages list + tools/storage.py get_experiment_history()

Claude Code (CLI) — The Builder

✓ USED — Built Everything

Claude Code (this tool, me!) designed the architecture, wrote all code, debugged issues, ran the pipeline, and generated reports. I used my own tools: Read, Write, Edit, Bash, Grep, Glob, Agent subagents. The entire project was built in one conversation.

Claude Code orchestrated the creation of the system it designed

MCP (Model Context Protocol)

✗ NOT USED

MCP allows Claude to connect to external tools/services via standardized servers (databases, APIs, file systems). Predicto calls APIs directly via Python. Future: could expose nba_api and Polymarket as MCP servers for real-time tool access.

Available but not needed — direct Python API calls suffice for PoC

Claude Agent SDK (claude_agent_sdk)

✗ NOT USED — Built From Scratch

Anthropic's Agent SDK provides higher-level abstractions (AgentRunner, handoffs, guardrails). We built our own minimal agent loop directly on the Messages API for full control. Future: migrating to Agent SDK would add built-in guardrails and handoff patterns.

We implemented a subset of Agent SDK patterns manually in base.py

Agent Handoffs / Delegation

~ PARTIAL

Agent SDK supports dynamic handoffs where one agent delegates to another mid-conversation. Predicto uses static sequential orchestration — main.py controls the flow. Agents don't call each other directly, but pass context via the orchestrator.

main.py passes summaries — not true dynamic handoffs

Subagents / Agent Teams

~ PARTIAL

Claude Code's "Agent" tool spawns subagents for parallel work. Predicto's agents are conceptually a team of specialists, but they run sequentially, not as Claude Code subagents. Future: Data + Feature agents could run in parallel.

Sequential team, not parallel subagents

Skills / Slash Commands

✗ NOT USED

Claude Code skills are reusable prompt templates invoked via /command. Predicto doesn't define custom skills. Future: could create /predict, /experiment, /report skills for quick access to pipeline stages.

Could wrap each agent as a Claude Code skill

Extended Thinking

✗ NOT USED

Claude's extended thinking mode enables deeper chain-of-thought reasoning before responding. Could improve Meta-Scientist's experiment design by letting it reason more carefully about which experiments to try next.

Would benefit Meta-Scientist's experimental design decisions

Prompt Caching

✗ NOT USED

The Anthropic API supports caching system prompts and tool definitions to reduce latency and cost on repeated calls. Since each agent makes 5-30 API calls with the same system prompt, caching would significantly reduce token costs.

Each agent's system prompt + tools are identical across iterations

Streaming

✗ NOT USED

The SDK supports streaming responses for real-time output. Predicto waits for complete responses. Future: streaming would let users see agent reasoning in real-time during long experiment runs.

Would improve UX for Meta-Scientist's long-running sessions

Hooks (Claude Code)

✗ NOT USED

Claude Code hooks run shell commands in response to events (pre-tool, post-tool). Could be used to auto-validate data after fetch, or trigger alerts when experiments complete. Not needed for this PoC.

Could trigger notifications on experiment completion
